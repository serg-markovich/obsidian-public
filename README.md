# 🧠 Sergey's DevOps Notes

My public knowledge base and learning journal — built with [Obsidian](https://obsidian.md),
published via [MkDocs Material](https://squidfunk.github.io/mkdocs-material/).

🌐 **Live site:** [notes.serg-markovich.de](https://notes.serg-markovich.de)

## What's inside

- **Azure** — AZ-104 prep notes, CLI commands, architecture patterns
- **DevOps** — CI/CD, Docker, Kubernetes, Infrastructure as Code
- **HomeLab** — VPS setup, self-hosted services, networking
- **Portfolio** — projects and case studies
- **TIL** — Today I Learned, quick findings and fixes
- **APPROACH** —  Daily development through an engineering lens

## Stack

| Tool | Purpose |
|---|---|
| Obsidian | Writing and linking notes |
| MkDocs Material | Static site generation |
| GitHub Actions | Automatic deploy on push |
| GitHub Pages | Hosting |
| Cloudflare | DNS |

## How it works

Notes are written in Obsidian, stored in this repo under `docs/`.
Every push to `main` triggers GitHub Actions which builds and deploys
the site to the `gh-pages` branch automatically.
